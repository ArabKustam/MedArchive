# ТЕХНИЧЕСКОЕ ЗАДАНИЕ

## Хакатон 2025

# MedPartners

## Автоматическая обработка архива прайсов клиник-партнёров и формирование базы услуг и цен

---

# 1. Цель и контекст

Создать систему автоматической обработки архива прайс-листов клиник-партнёров, которая:

- Разбирает входящие документы различных форматов (PDF, DOCX, XLSX, сканы)
- Извлекает структурированные данные: клиника, услуга, цена (с учётом тарифов резидент/нерезидент)
- Нормализует данные, приводя их к единому целевому справочнику услуг
- Формирует верифицированную базу данных партнёров, услуг и цен
- Предоставляет API и интерфейс для поиска: кто оказывает услугу и по какой цене

Целевой справочник медицинских услуг будет предоставлен организаторами в момент старта хакатона. Команды должны предусмотреть механизм загрузки справочника и привязки к нему всех извлечённых услуг.

---

# 2. Входные данные

## 2.1 Архив прайс-листов

Организаторы предоставляют архив (ZIP) с прайсами клиник-партнёров.

Архив содержит документы следующих типов:

| Тип файла | Описание | Сложность извлечения |
|-----------|----------|----------------------|
| PDF (из DOCX) | Структурированный текст, таблицы читаются корректно | Низкая |
| PDF (скан) | Изображение документа, требуется OCR | Высокая |
| XLSX / XLS | Табличные данные, возможны множественные листы | Средняя |
| DOCX | Текстовые документы с таблицами | Средняя |

Один файл = прайс одной клиники-партнёра на конкретную дату.

Имя файла может содержать название клиники и/или дату.

Для одного партнёра может быть несколько прайсов — необходимо учитывать все версии и строить историю изменения цен.

---

## 2.2 Целевой справочник услуг

Предоставляется организаторами в формате **XLSX** или **JSON**.

Минимальный состав:

| Поле | Тип | Описание |
|------|-----|----------|
| service_id | uuid | Уникальный идентификатор услуги |
| service_name | string | Официальное название |
| synonyms | array<string> | Список синонимов |
| category | string | Категория услуги |

# 4. Функциональные требования

## 4.1 Загрузка архива

- Приём ZIP через интерфейс или CLI
- Автоматическое определение типа файлов
- Очередь обработки
- Сохранение оригиналов

---

## 4.2 Извлечение данных

### PDF (текст)

- Извлечение текста
- Извлечение таблиц
- Распознавание структуры

### PDF (скан)

- OCR (Tesseract, Google Vision, AWS Textract)
- Очистка результатов OCR

### XLSX / XLS

- Обход всех листов
- Поиск строки заголовков
- Извлечение услуг и цен

### DOCX

- Принятие tracked changes
- Извлечение таблиц

---

## 4.3 Нормализация

- Точное совпадение
- Поиск по синонимам
- Нечёткий поиск

Порог уверенности:

- cosine similarity > 0.85 → автоматически
- иначе → очередь проверки

Несопоставленные позиции отправляются в очередь `unmatched`.

---

## 4.4 Валидация

Автоматические проверки:

| Проверка | Действие |
|----------|----------|
| Цена > 0 | needs_review |
| Цена нерезидента >= резидента | Предупреждение |
| Пустое название | Пропуск |
| Дата не из будущего | Предупреждение |
| Дубликат | Архивировать старую |
| Изменение цены >50% | Флаг аномалии |
| Валюта ≠ KZT | Конвертация |
| Не удалось распознать | error |

Ручная проверка:

- Просмотр документа
- Проверка данных
- Подтверждение
- Исправление
- Версионирование

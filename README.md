# MedArchive — Система обработки прайс-листов и поиска медицинских услуг

**MedArchive** — это современная система для автоматической обработки, нормализации и аналитики медицинских прайс-листов клиник-партнёров. Система позволяет принимать документы в любых форматах (PDF, Excel, Word, сканы, ZIP-архивы), автоматически распознавать таблицы и текст, сопоставлять позиции с эталонным каталогом услуг и предоставлять удобный интерфейс для поиска, сравнения цен и модерации.

---

## 🚀 Быстрый запуск

### Вариант 1. Запуск в один клик

**Для macOS / Linux:**
* `./run.sh` или `./start.sh` — автоматически запускает сервер бэкенда и веб-интерфейс фронтенда.
* `./stop.sh` — корректно останавливает работу всех сервисов приложения.

**Для Windows:**
* `run.bat` или `start.bat` — запуск всех сервисов.
* `stop.bat` — остановка серверов.

---

### Вариант 2. Ручной запуск (Разработка)

#### 1. Запуск Бэкенда (Python / FastAPI)
```bash
# 1. Перейдите в папку бэкенда
cd backend

# 2. Создайте и активируйте виртуальное окружение (опционально)
python -m venv .venv
# На Windows:
.venv\Scripts\activate

# 3. Установите зависимости
pip install -r requirements.txt

# 4. Запустите сервер Uvicorn
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8765
```
> Бэкенд будет доступен по адресу: `http://localhost:8765`  
> Интерактивная автодокументация Swagger (OpenAPI): `http://localhost:8765/docs`

#### 2. Запуск Фронтенда (React / Vite)
```bash
# 1. Находясь в корневой папке проекта, установите зависимости (через bun или npm)
npm install
# или: bun install

# 2. Запустите dev-сервер Vite
npm run dev
# или: bun dev
```
> Фронтенд будет доступен по адресу: `http://localhost:5173`

## 🛠 Технологический стек

### 🎨 Frontend (Клиентская часть)
* **React 19** — библиотек интерфейсов с поддержкой современных хуков и серверных компонентов.
* **TypeScript 5.8** — строгая типизация данных.
* **Vite 8** — быстрый сборщик и dev-сервер.
* **Feature-Sliced Design (FSD)** — архитектурная методология организации кода по слоям (`pages`, `widgets`, `features`, `entities`, `shared`).
* **TanStack Router** — типобезопасный файловый роутинг.
* **TanStack React Query** — кэширование и фоновая синхронизация API-запросов.
* **Tailwind CSS 4 & Radix UI** — адаптивный дизайн и доступные UI-компоненты.
* **Recharts** — интерактивные графики и визуализация аналитики.

### ⚙️ Backend (Серверная часть)
* **Python 3.11+** — основной язык разработки.
* **FastAPI** — высокопроизводительный асинхронный REST API фреймворк.
* **Uvicorn** — боевой ASGI-сервер.
* **SQLAlchemy 2.0 & SQLite** — ORM для работы с базой данных `medarchive.db` в режиме WAL.
* **Pydantic v2** — строгая валидация входящих и исходящих структур данных.

### 📄 Парсинг и Нормализация
* **Разбор файлов:** `pdfplumber` (PDF), `python-docx` (Word), `python-calamine` & `openpyxl` (Excel), `zipfile` (распаковка архивов).
* **OCR (Оптическое распознавание):** Google Cloud Vision API / `pytesseract` для извлечения текста со сканов документов.
* **LLM Extraction:** Интеграция с Groq API / LLaMA для сложных неструктурированных файлов.
* **RapidFuzz:** Алгоритмы нечёткого поиска (Fuzzy Matching) для сопоставления услуг с вычислением индекса уверенности (`confidence score`).

## 📁 Структура проекта

```
pixel-perfect-clone-37943/
├── backend/                  # Серверное приложение на Python (FastAPI)
│   ├── app/
│   │   ├── main.py           # Точка входа API и настройка CORS
│   │   ├── db.py             # Подключение SQLAlchemy и сессии БД
│   │   ├── models.py         # Модели таблиц базы данных
│   │   ├── parsers.py        # Извлечение таблиц из PDF/XLSX/DOCX и OCR
│   │   ├── normalize.py      # Алгоритмы сопоставления со справочником
│   │   ├── pipeline.py       # Фоновый пайплайн обработки файлов
│   │   └── routers/          # Маршруты API (upload, search, services, partners, verification, stats)
│   ├── requirements.txt      # Зависимости Python
│   └── run_backend.bat       # Скрипт запуска бэкенда
│
├── src/                      # Клиентское приложение на React (FSD Архитектура)
│   ├── pages/                # Страницы (Поиск, Дашборд, Загрузка, Верификация, Списки)
│   ├── widgets/              # Крупные блоки (AppLayout, навигация)
│   ├── features/             # Пользовательские сценарии (загрузка, удаление, просмотр)
│   ├── entities/             # Бизнес-сущности (Услуга, Партнёр, Документ)
│   └── shared/               # Переиспользуемый UI, API-клиент, типы
│
├── run.bat / start.bat       # Единый скрипт запуска всего проекта
├── ARCHITECTURE.md           # Подробное техническое описание архитектуры
└── package.json              # Зависимости и скрипты Node.js / Vite
```
